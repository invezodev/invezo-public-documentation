---
title: "Profit Loss"
description:
  "Get approximate P&L in USD of a wallet address over a specified time period based on balance. An empty token parameter will give an aggregate P&L over time in USD for all tokens."
api: "GET https://api.invezo.com/v1/wallets/tracking/balance"
---

### Query Parameters

<ParamField query="chain" type="string" required>
  platform to query (valid chains: eth, sol)
</ParamField>

<ParamField query="min" type="int or string" required>
  start of the aggregate time window as a timestamp (example: 1652486400 or '2022-05-13T18:00:00')
</ParamField>

<ParamField query="max" type="int or string" required>
  end of the aggregate time window as a timestamp (example: 1675191600 or '2023-01-31T12:00:00')
</ParamField>

<ParamField query="interval" type="string" required>
  size of the time window (valid intervals: 10m, 1h, 1d, 1w, 1mo)
</ParamField>

<ParamField query="token" type="string" optional>
  asset contract address (example: EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v, 0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48)
</ParamField>

<ParamField query="sort" type="string" optional>
  sort the results by timestamp. asc will return results in ascending order
  (oldest to newest), desc will return results in descending order (newest to
  oldest). default value is asc
</ParamField>

<ParamField query="limit" type="int" optional>
  limit the number of results returned, default is 5000 and max is 50000
</ParamField>
